# API Reference

Base URL: `http://localhost:3000` (local) or the deployed Railway URL.

All error responses follow this format:
```json
{
  "error": "ERROR_CODE",
  "message": "Human-readable description."
}
```

---

## POST /verify

Verifies a ZK location proof and issues a JWT token.

**Rate limit:** 10 requests/minute per IP.

### Request

```json
{
  "proof": "<hex-encoded UltraHonk proof>",
  "public_inputs": {
    "nullifier_hash": "<32-byte hex>",
    "region_id": "<16-byte hex>",
    "centroid_lat": 44787000,
    "centroid_lon": 20457000,
    "radius_m": 1000,
    "slot_field": "454344535"
  },
  "expires_in_seconds": 3600
}
```

| Field | Type | Description |
|---|---|---|
| `proof` | string | Hex-encoded UltraHonk proof generated by the circuit |
| `public_inputs.nullifier_hash` | string | 32-byte hex — unique per proof, prevents replay |
| `public_inputs.region_id` | string | 16-byte hex — must match a registered region on-chain |
| `public_inputs.centroid_lat` | number | Region centroid latitude in micro-degrees (e.g. 44.787° = 44787000) |
| `public_inputs.centroid_lon` | number | Region centroid longitude in micro-degrees |
| `public_inputs.radius_m` | number | Region radius in metres |
| `public_inputs.slot_field` | string | Current Solana slot as decimal string — must be within ±150 slots (~60s) |
| `expires_in_seconds` | number | Optional. JWT expiry 1–3600 seconds. Default: 3600 |

> `slot_field` is a decimal string (not a number) to avoid JavaScript precision loss on u64 values.

### Response `200 OK`

```json
{
  "jwt": "<signed JWT token>",
  "expires_at": 1712345678
}
```

### JWT Claims

```json
{
  "sub": "<nullifier_hash>",
  "region_id": "<hex>",
  "region_name": "Beograd centar",
  "zk_verified": true,
  "solana_slot": "454344535",
  "vk_version": "1",
  "iat": 1712342078,
  "exp": 1712345678
}
```

### Error Codes

| Code | Status | Description |
|---|---|---|
| `INVALID_INPUTS` | 400 | Missing or malformed fields |
| `SLOT_EXPIRED` | 400 | `slot_field` is older than 150 slots (~60s) |
| `SLOT_IN_FUTURE` | 400 | `slot_field` is ahead of current slot |
| `REGION_NOT_FOUND` | 404 | Region not registered on-chain |
| `REGION_MISMATCH` | 400 | Public inputs don't match on-chain RegionAccount |
| `PROOF_INVALID` | 400 | ZK proof verification failed |
| `NULLIFIER_USED` | 409 | This nullifier has already been registered |
| `SERVICE_UNAVAILABLE` | 503 | Circuit not loaded or Solana unreachable |

## GET /recover

Re-issues a JWT for an existing nullifier. Used when a JWT has expired but the nullifier is still within the 90-minute recovery window.

**Rate limit:** 5 requests/minute per IP.

### Authentication

The request must include an Ed25519 signature proving ownership of the nullifier:

```
Authorization: Bearer <64-byte hex signature>
```

The signature is computed over the raw bytes of `nullifier_hash` using the session Ed25519 private key.

### Request

```
GET /recover?nullifier_hash=<32-byte hex>&public_key=<32-byte hex>
```

| Parameter | Description |
|---|---|
| `nullifier_hash` | 32-byte hex — the nullifier to recover |
| `public_key` | 32-byte hex Ed25519 public key of the session keypair |

### Response `200 OK`

```json
{
  "jwt": "<signed JWT token>",
  "expires_at": 1712345678
}
```

### Error Codes

| Code | Status | Description |
|---|---|---|
| `INVALID_INPUTS` | 400 | Missing parameters or malformed signature format |
| `UNAUTHORIZED` | 401 | Signature verification failed |
| `REGION_NOT_FOUND` | 404 | Nullifier not found on-chain |
| `NULLIFIER_USED` | 409 | Nullifier is outside the 90-minute recovery window |
| `SERVICE_UNAVAILABLE` | 503 | Solana unreachable |

## GET /regions/nearby

Returns regions near a given location.

**Rate limit:** 60 requests/minute per IP.

### Request

```
GET /regions/nearby?lat=44787000&lon=20457000
```

Coordinates are in **micro-degrees** (decimal degrees × 1,000,000).

### Response `200 OK`

```json
[
  {
    "region_id": "3fe6cd7e9924085c792e7974402fb620",
    "name": "Beograd centar",
    "centroid_lat": 44787000,
    "centroid_lon": 20457000,
    "radius_m": 5000,
    "distance_m": 411
  }
]
```

Results are sorted by distance. If multiple regions share the same name, only the most precise one is returned.

## GET /slot

Returns the current Solana slot. Used by the UI to construct `slot_field` for the proof.

### Response `200 OK`

```json
{
  "slot": "454344535",
  "timestamp_ms": 1712342078000
}
```

> Response includes `Cache-Control: no-store` to prevent browser caching.

---

## GET /jwks

Returns the JSON Web Key Set for verifying issued JWTs. Third-party applications can use this to verify JWT signatures independently.

### Response `200 OK`

```json
{
  "keys": [
    {
      "kty": "EC",
      "crv": "P-256",
      "alg": "ES256",
      "use": "sig",
      "kid": "d2a17111ab0be9ca",
      "x": "...",
      "y": "..."
    }
  ]
}
```

> Response includes `Cache-Control: public, max-age=3600`.

---

## GET /vk/:version

Returns the verification key for the current circuit version. Used by integrators who want to verify proofs independently.

### Request

```
GET /vk/1
```

### Response `200 OK`

```json
{
  "version": "1",
  "vk_hex": "<hex-encoded verification key>",
  "circuit_hash": "<SHA-256 of circuit.json>"
}
```

## GET /health

Returns the current status of the backend services.

### Response `200 OK` (all services ready)

```json
{
  "status": "ok",
  "verifier": true,
  "jwtSigner": true,
  "timestamp_ms": 1712342078000
}
```

### Response `503 Service Unavailable` (degraded)

```json
{
  "status": "degraded",
  "verifier": false,
  "jwtSigner": true,
  "timestamp_ms": 1712342078000
}
```
